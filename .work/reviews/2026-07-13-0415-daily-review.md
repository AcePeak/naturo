## What the mask was hiding

The headline is not the count. It is **what** was in it.

**#1291 (P0, filed) — a Never-Lie violation on the primary input path.** `type_text` and `press_key` return `success: true` when `window_title` does not resolve, and the keystrokes land in **whatever window happens to be focused**. That is a wrong-window *write* — destructive, not merely incorrect. The guard that catches it (`test_mcp_window_selector_contract_957`, the self-maintaining contract built by #957 to make this bug class unshippable) **was already red and doing its job.** CI was discarding its verdict. Detection was never the problem; the reporting channel was.

**#1292 (P1, filed) — the MCP surface has silently diverged from the CLI.** `app_hide`, `app_unhide`, `hotkey`, `list_snapshots`, `get_snapshot` all exist in the codebase but are **not registered as MCP tools**. This is Pillar B (agent fit), not tech debt: a capability naturo has on the CLI but not over MCP is a capability an agent *cannot use*. Our own surface-coverage guard (#912) was saying so, into a muted channel.

**#1293 (P1, filed) — MCP error-contract and see→click schema regressions.** Input-validation failures surface to agents as `INTERNAL_ERROR` instead of `INVALID_INPUT` (an agent cannot self-correct from `INTERNAL_ERROR`), and `see_ui_tree`'s response drops the `tree` key, breaking the `see` → `eN` → `click` round-trip — the primary agent loop. Some of these may be tests made legitimately stale by #1289's depth change; each needs a real triage decision, not a delete.

**#1290** (mine, merged `648aa99`): `tests/conftest.py`'s CI-Windows guard replaces two constructors — `_SkippingWindowsBackend` **subclasses** the real backend, but `_SkippingNaturoCore` was a **bare class**. `NaturoCore` defines 34 class-level methods, so every test that introspects the class (signature checks, `mock.patch.object`, `MagicMock(spec=NaturoCore)`) hit a stub with none of them and raised `AttributeError` instead of skipping. Making it subclass the real core — mirroring the sibling that was already correct — cleared **39** tests. `__init__` still short-circuits before `self._load(...)`, so no DLL loads and no desktop is touched; the guard's actual purpose is intact.

Verified the way the rule requires: read on the real Windows job, `63 → 24`, zero `_SkippingNaturoCore` errors left, all other jobs green. Not auto-merged — the Windows job is still non-blocking, so it would have merged green whether or not the fix worked.

**All 24 remaining are attributed: #1291 (2) + #1292 (11) + #1293 (11) + #1175 (1, known flake) = 24.** Close those four and #1274 lands, permanently closing the hole.
