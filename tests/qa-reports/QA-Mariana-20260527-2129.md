### Rejected ❌ (status:done removed, returned to Dev)
- **#844 MCP Pydantic leak** — fix is dead code. Root cause documented in detail on the issue: PR #853 reassigns `server.call_tool = _sanitized_call_tool` AFTER `FastMCP.__init__` has already run `_setup_handlers` (mcp/server/fastmcp/server.py:239,308). The low-level `_mcp_server` captured the original bound method at instantiation; the late reassignment is invisible to the dispatcher. Reproduced verbatim Pydantic-laden error message ("Error executing tool launch_app: 1 validation error for launch_appArguments... pydantic.dev/2.12/v/missing"). Unit tests pass because they call `await server.call_tool(...)` directly, bypassing the real JSON-RPC dispatch path. Suggested fix: patch the registration itself (subclass FastMCP, override `_mcp_server.request_handlers[CallToolRequest]`, or wrap `Tool.run` at mcp/server/fastmcp/tools/base.py:117 where the raw message is constructed).

## Phase 4 — Accessibility User Simulation
Persona perspective: keyboard-only navigation, screen-reader friendliness.

**Strengths**:
- Button names include keyboard shortcut hints inline ("加粗(Ctrl+B)", "斜体(Ctrl+I)", "链接(Ctrl+K)", "清除格式设置(Ctrl+空格)") — excellent for screen-reader announcement.
- CLI primitives for keyboard-only flows: `--on eN`, `--selector`, `--tab N`, `--return`, `--escape` — solid coverage.
- `naturo see` exposes both `role` and `name` for every visible UI element — the foundation a screen-reader script needs.

**Gaps**:
- Of 44 elements in fresh Notepad, 13 (29%) are unnamed; most are structural `Pane` containers which is normal, but some `Pane` and `Window` nodes (e28-e30 offscreen panes for "Local AI model downloading…") would be confusing in a flat list.
- `naturo --help` makes no explicit mention of accessibility / keyboard-only workflows. A keyboard-only user discovering the tool via `--help` wouldn't find the discoverability path.
- Text format prints `eN`; JSON field is `id`. Scripted accessibility tooling has to remember the mapping. Not severe enough to file.
