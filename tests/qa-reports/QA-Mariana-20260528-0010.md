## Phase 4 — First-time User Simulation
Adopted the mindset of someone who just `pip install naturo`'d and is reading the README for the first time. Friction observed:

1. **Hidden `snapshot` command leaks via Did-you-mean (FILED #867)** — `naturo --help` deliberately omits `snapshot` (its own description says "internal/debug command"), but Click's typo engine suggests it for `screenshot`, `snap`, `snapshots`. A first-time user looking for screenshot functionality follows the suggestion straight into an internal debug group and is misled. The real answer is `naturo capture` (or `see`).
2. **README quick-start examples generally line up with `--help`** — verified `naturo see --window`, `naturo capture --path`, `naturo type "..."` flags all exist. Good. (No bug here this round.)
3. **`naturo help` works as a happy accident** — returns help text and exit 0, even though it's not documented. Surprisingly forgiving for first-time users. Keep.
4. **`naturo see notepad` (positional argument) gives "unexpected extra argument"** — a newcomer's natural instinct (it feels like `git checkout notepad`) is rejected without a hint that they meant `--app notepad`. Not a bug, but a missed-opportunity-for-friendliness. Not filing this round.
5. **MCP setup path is clean** — `naturo mcp install` works, `naturo mcp tools` lists 64 tools with proper `-j` envelope. Good for AI-agent-builders following the README.

The one concrete bug from this persona: **#867** (snapshot leak).
