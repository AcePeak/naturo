## Phase 4 — Power User simulation
A Power User with 10+ apps + multi-monitor + 150% DPI was infeasible to fully simulate from this no-desktop session. Adapted by attacking the parts of the surface a Power User *does* exercise heavily — MCP, JSON pipelines, `--app` filter edge cases — and found two new bugs:

### #872 (NEW, P2) — Click UsageError bypasses `-j` JSON envelope
`-j`/`--json` is honoured by naturo's own validation (e.g. `see --depth -1 -j` → envelope) but Click-level argument errors (unknown option, invalid type) emit plain text and exit 2. Affects every subcommand tested (see, click, capture, type, press, list). Position of `-j` (before or after the subcommand) doesn't change behaviour. The missing-argument path was fixed by #123; the unknown-option path was never extended. Distinct from #865/#869.

```
$ naturo see --depth abc -j | jq .
parse error: Invalid numeric literal at line 1, column 6
```

### #873 (NEW, P2) — MCP serverInfo.version reports mcp SDK version
`naturo mcp start --transport stdio` advertises `serverInfo.version = "1.26.0"` — the version of the `mcp` Python package, not naturo (`0.3.1`). Misleading for any AI agent that branches on server version for capability detection. Likely a missing `version=` kwarg on `FastMCP()` / `Server()` construction.
