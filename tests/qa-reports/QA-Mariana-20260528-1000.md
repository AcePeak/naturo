## Phase 4 — Enterprise RPA Dev simulation
Ran a 10-step reconciliation workflow (`launch → see → find → type → press tab → type → press enter → capture → get → quit`) scripted with `-j` and per-step exit-code capture (`/tmp/r130-rpa-workflow.sh`). Every step exited 1 with a structured envelope; no step bombed with exit 2 or empty output. Notable RPA-perspective observations:

1. **`-j` mode contract is reliable enough for `set -e` scripts.** Mixed mode (some steps `-j`, some not) is not — non-`-j` input commands exit 2 (#866), which `set -e` treats the same as exit 1.
2. **`UNKNOWN_ERROR` (#877) is the only generic code observed mid-flow.** Enterprise dispatchers want `STALE_ELEMENT_REF` so they can branch `if (err.code == STALE_ELEMENT_REF) refresh_snapshot(); retry;`. Matching on message strings is fragile.
3. **Chain recovery would work.** Step 10 `app quit notepad` after launch failure returned `APP_NOT_FOUND` cleanly — the chain is restartable.
4. **Error messages embed remediation hints.** `NO_DESKTOP_SESSION` includes `suggested_action` pointing at RDP/Console, which is the right pattern for ops staff but is missing for `UNKNOWN_ERROR` (#877).
