## Phase 3 — Professional Exploratory Testing
Probed the CLI JSON error envelope across **13 commands** under both NO_DESKTOP_SESSION and stale-snapshot-ref error paths. Discovered three distinct envelope shapes for the same `{success:false, error:{...}}` contract:

| Shape | Fields | Commands |
|------|--------|---------|
| **A (rich, 6)** | code, message, category, context, suggested_action, recoverable | `app launch`, `app quit`, `app focus`, `app maximize` |
| **B (flat, 3)** | code, message, suggested_action | `see`, `capture`, `list apps/windows/screens`, `type`, `press`, `click`, `find` |
| **C (minimal, 2)** | code, message | `get`, `set` (UNKNOWN_ERROR path; partial overlap with #877) |

Root cause: most commands route through `naturo/cli/error_helpers.py:225 json_error()`, which strips `category` / `context` / `recoverable` even though `NaturoError.to_json_response()` (`naturo/errors.py:106`) emits the full schema and is used by the `app *` family. Filed **#884** (P2, v0.3.4).
