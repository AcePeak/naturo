## Phase 4 — Scripter/Automator findings

**1. `record show|delete|export|play -j` return `error` as plain STRING** (new — shape D of #884)
A Scripter doing `jq '.error.code'` on these endpoints gets `null` because `.error` isn't an object. Strictly worse than shapes A/B/C tracked in #884.
```json
{"success": false, "error": "Recording not found: bogus_id_999"}
```
Filed as comment on #884.

**2. `visual list -j` also missing top-level `success` field** (extension of #876)
`#876` originally listed `selector list -j` (returns `{}`) and `record list -j` (returns `{"recordings": [...]}`). Adding `visual list -j` → `{"baselines": []}`. Same root cause; one-line fix per callsite.
Filed as comment on #876.

**3. `browser launch -j` error path uses `success: false` but exits 0** (extension of #879)
The original #879 was about the success path returning `{status: ok}`. R134 finds the **error path** now uses the correct `{success: false, error: {...}}` envelope but still exits 0 — so `if naturo browser launch -j; then` evaluates true even on failure. Script-breaking.
Filed as comment on #879.

**4. Optional-dep install prompt suppression depends on stdin TYPE, not just isatty** (nuance to #869)
- `naturo desktop list -j </dev/null` → prompt leaks before JSON envelope
- `echo '' | naturo desktop list -j` → prompt suppressed, clean JSON envelope
Same `_is_interactive()` evaluating to False in both cases but different stdout content. Subtle Windows-specific path. Filed in TC-0057 notes.

**5. Confirmed silent-failure surface in dialog/taskbar/tray** (#875)
`naturo dialog detect -j`, `naturo taskbar list -j`, `naturo tray list -j` all return `{"success": true, "<items>": [], "count": 0}` exit 0 in NO_DESKTOP_SESSION. These are exactly the kind of silent-success bugs epic #885 was filed to eradicate, but they're not currently in the epic's scope. **Recommendation for orc:** consider folding #875 into #885 alongside the existing #868/#878/#883.

**6. Confirmed: `naturo app windows -j` bypasses NO_DESKTOP_SESSION guard** (#878)
Returns real Chrome window data (including Chinese title `要恢复页面吗？`) in a session where every other read command refuses. Same #885 bucket.
