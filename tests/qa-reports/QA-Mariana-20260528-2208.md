### Finding: missing-required-arg exit code drift (filed as #897)

A Scripter's standard dispatcher:
```bash
naturo type 2>/dev/null
case $? in
  0) echo SUCCESS ;;
  1) echo "OPERATION_FAILED (retry?)" ;;
  2) echo "USAGE_ERROR (don't retry)" ;;
esac
```

…misclassifies `naturo type` (missing TEXT) as `OPERATION_FAILED` and infinite-retries.

Cause: two argument-validation paths return different exit codes for the same *class* of failure:

| invocation                  | exit | path                         |
|-----------------------------|------|------------------------------|
| `naturo nonexistentcmd`     | 2    | Click default                |
| `naturo see --app` (no val) | 2    | Click missing-flag-value     |
| `naturo` (no subcmd)        | 2    | Click group help             |
| `naturo app` (no subcmd)    | 2    | Click group help             |
| `naturo type`               | **1**| naturo INVALID_INPUT custom  |
| `naturo press`              | **1**| naturo INVALID_INPUT custom  |
| `naturo find`               | **1**| naturo INVALID_INPUT custom  |
| `naturo wait`               | **1**| naturo INVALID_INPUT custom  |
| `naturo get`                | **1**| naturo INVALID_INPUT custom  |
| `naturo set`                | **1**| naturo INVALID_INPUT custom  |
| `naturo app launch`         | **1**| naturo INVALID_INPUT custom  |

This is the *argument-validation* axis of the exit-code contract; **distinct from #866** which is the *NO_DESKTOP-runtime* axis. Together they mean **no single exit code today can be trusted by a script for usage-vs-runtime classification**.

Filed as **#897** (P2 v0.3.4). Suggested fix: bind `INVALID_INPUT` to `sys.exit(2)` in the custom validators; keep `INVALID_INPUT` envelope code unchanged.
