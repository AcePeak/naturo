**Key first-time observations**:

1. **`-h` not accepted anywhere** is the loudest first-30-seconds papercut. Every Click-based CLI a first-time user has touched (`git -h`, `curl -h`, `pip -h`, `python -h`, `ls -h`) accepts it. The README and `--help` body do not document the asymmetry. Trivial one-line root-group fix via `context_settings={'help_option_names': ['-h', '--help']}` propagates to all subcommands. → **#899** (P2 v0.3.4).
2. **Did-You-Mean leaks `help` as a 3rd hidden command** (after `snapshot` R136, `hotkey` R136). Extension comment on #867 — same one-line fix in `FuzzyGroup` suggester covers all three.
3. **Input commands' "Usage:" banner on runtime errors** is especially confusing for a first-time user because the user has no mental model for "this is an environmental error vs an argument error" — the Click banner trains them to go re-read `--help`, which doesn't tell them anything useful. (#866, unchanged but reinforced from First-time User lens.)
4. **Silent successes on `dialog detect` / `taskbar list` / `tray list`** mean a first-time user piping these to a script thinks "OK, no taskbar items" — when in fact naturo couldn't even ask. The most dangerous failure mode in the entire CLI surface (#875).
5. **`naturo config show` is the highest-quality first-time-UX command in the CLI.** No `naturo doctor` yet (#898), but `config show` already nails the shape that diagnostic should aspire to.
