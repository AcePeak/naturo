### #868 sub-finding ‚Äî out-of-range `screen_index` also silent-captures

`list_monitors` reports only index 0. `capture_screen` calls with `screen_index ‚àà {0, 1, 99}` all return success:true and a 24,219-byte 3840√ó2160 PNG with PIL extrema `((0,0),(0,0),(0,0))` (100% black). Only `screen_index=-1` errors (`INTERNAL_ERROR: NaturoCoreError: capture_screen: Invalid argument`).

Metadata is misleading: index 0 ‚Üí `scale 1.5 / dpi 144`; indices 1 and 99 ‚Üí `scale 1.0 / dpi 96`. An agent reading metadata thinks two distinct monitors were sampled. **Comment posted to #868.**

### NEW BUG: MCP window-targeting param name drift ‚Äî **#900 filed**

Three names for the same logical parameter:
- CLI: `--window`
- MCP `_window.py` family (focus_window, window_close, ‚Ä¶): `title`
- MCP `_capture.py`, `_inspect.py`, `_snapshot.py`, `_wait.py`: `window_title`

Compounds with #891: an agent translating `naturo app focus --window "Chat"` to MCP `focus_window({"window":"Chat"})` gets `{success:false, error.code:"WINDOW_NOT_FOUND", message:"Window not found: foreground"}` ‚Äî `window` is silently dropped, `title=None` defaults to "foreground", error message points nowhere useful.

Filed: **#900** (P2, v0.3.4, from:qa). Test case **TC-0085** added.
