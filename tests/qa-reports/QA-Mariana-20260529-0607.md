### New / sharper findings this round
1. **#878 — intra-`app`-group inconsistency:** `app list -j` correctly refuses (exit 1) while its sibling `app windows -j` (and non-json) bypasses. The split is inside one command group, not just across groups. Both modes bypass.
2. **#893 — verifiably false negative (not just "empty"):** `naturo wait --gone explorer -j` → `success:true, found:true` while the same-session guard message names **explorer.exe as running**. Prior repros used absent selectors; this pins it as fail-open, not under-information. Workflow chain `wait --gone … && echo PROCEEDING && click …` prints PROCEEDING (wait_time 0, `--timeout` ignored) then fails one step late. Commented on #893.
3. **#869 refinement:** `desktop list -j` only shows the interactive pyvda `Install it now? [Y/n]` prompt when stdin is a **TTY**. Under piped/non-interactive stdin (real script context) it returns a clean `{"success":false,"error":{"code":"MISSING_DEPENDENCY"}}` exit 1. So #869 is an interactive-only cosmetic leak, **not** a script-hang. (Noted for triage; no scope change.)
